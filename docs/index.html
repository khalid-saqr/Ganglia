The existing pain

LLM apps fail at the seam between intention and output.

Developers do not only need fluent answers. They need outputs that arrive in a known shape, follow a declared reasoning role, survive checks, and leave an audit trail.

The philosophical turn

Wittgenstein, translated for LLM middleware.

In Philosophical Investigations, Wittgenstein develops language-games to show that meaning lives in use: words function inside practical activities, with roles, rules, expectations, and criteria for whether a move makes sense.

Ganglia adapts that design lens. A model answer is not treated as a blob of text. It is treated as a move inside a declared activity that can be checked before release.

SituationWhat kind of work is the user asking for?
RulesWhat must count as a valid move?
ObjectWhat structured artifact must be returned?
CriteriaHow do we detect broken, missing, or repaired moves?

Ganglia's thesis

Do not try to read the model's mind. Control the public artifact.

Ganglia does not claim to expose hidden chain-of-thought or control internal model cognition. It controls what developers can observe and test: selected operator, compiled prompt, required JSON schema, semantic validation, repair attempts, renderer, and traceable final answer.

Reasoning design pattern

A prompt becomes a protocol when every move has a gate.

1

Route

Select an operator automatically or by model name.

2

Constrain

Compile rules, prompt template, and JSON Schema into the model call.

3

Validate

Check both shape and semantics before accepting the answer.

4

Repair

Send failed outputs back through an operator-specific repair prompt.

5

Release

Render the controlled object and store a trace.

Built-in language-games

Three public reasoning moves, not three hidden thoughts.

Spatial judgment

coordinate_game

Forces abstract entities into a two-axis 0-10 coordinate system.

  • mapping
  • plotting
  • positioning
  • scoring

Comparative judgment

grid_game

Forces content into a comparative matrix.

  • comparison
  • classification
  • MECE table forcing
  • relational ontologies

Adversarial judgment

adversarial_grid_chain

Applies adversarial testing, then formalises the result as a failure matrix.

  • stress-testing
  • falsification
  • red-team review
  • risk detection

Runtime shape

A local gateway that tools can call like a model.

Ganglia exposes a native reasoning endpoint and an OpenAI-compatible chat completions endpoint, so existing tools can call a controlled reasoning surface instead of talking directly to the LLM backend.

Reference-grounded claims

What this page is, and is not, claiming.

The page uses Wittgenstein's language-game idea as a public design explanation for Ganglia's operators. It does not claim that LLM internals are readable or fully controllable. The controlled surface is the response artifact that can be validated, repaired, rendered, traced, and returned.
